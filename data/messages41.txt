ai streszczenie tego co rozmowy dotyczy <br><br>- Open Addressing: If a collision occurs, the hashmap searches for the next available empty slot in the same array to store the data. Linear probing is one such technique where it searches sequentially. During lookup, the hash is calculated, the index is checked, and if the key doesn&apos;t match, it continues to search linearly until the key is found or an empty slot is encountered<br><br>- Closed Addressing: Instead of storing the value directly in the array, each slot in the array might store a pointer to a collection (like a linked list) where the actual key-value pairs with the same initial hash index are stored. When a collision occurs, the new key-value pair is added to the collection at that index.<br><br>With a good hash function that distributes keys evenly and a low load factor (how full the hashmap is), hashmap operations like insertion, deletion, and lookup can ideally be close to constant time, O(1). This is because the hash function allows direct access to the approximate location of the data.<br><br>However, the video highlights that collisions can degrade performance. In the case of open addressing with linear probing, if there are many collisions leading to clustering (long runs of filled slots), the time it takes to find an empty slot for insertion or the desired key for lookup can approach linear time, O(n), in the worst case (if the hashmap is nearly full)<br><br>In closed addressing with linked lists, in the worst-case scenario where many keys hash to the same index, the lookup within the linked list could take linear time, O(n), where n is the number of key-value pairs in that particular linked list (or in the entire hashmap in the very worst distribution)<br><br>Resizing the underlying array, which is needed when the load factor becomes too high to avoid excessive collisions, involves allocating a new, larger array and then re-hashing and re-inserting all the existing elements. This operation can take O(n) time, where n is the number of elements in the hashmap
